一、Java简介

编译型语言
如C、C++，代码是直接编译成机器码执行，但是不同的平台（x86、ARM等）CPU的指令集不同，因此，需要编译出每一种平台的对应机器码。

解释型语言
如Python、Ruby没有上述问题，可以由解释器(compiler)直接加载源码然后运行，代价是运行效率太低
Java介于编译型语言和解释型语言之间
Java是将代码编译(compiler)成一种“字节码”，它类似于抽象的CPU指令，然后，针对不同平台编写虚拟机，不同平台的虚拟机负责加载(解释器 interpreter)字节码并执行，这样就实现了“一次编写，到处运行”的效果。当然，这是针对Java开发者而言。

对于虚拟机，需要为每个平台分别开发。为了保证不同平台、不同公司开发的虚拟机都能正确执行Java字节码，SUN公司制定了一系列的Java虚拟机规范。
从实践的角度看，JVM的兼容性做得非常好，低版本的Java字节码完全可以正常运行在高版本的JVM上。

Java虚拟机

打比方

随着Java的发展，SUN给Java又分出了三个不同版本：

  • Java SE：Standard Edition
  • Java EE：Enterprise Edition
  • Java ME：Micro Edition
这三者的关系：

Java SE就是标准版，包含标准的JVM和标准库（核心）
Java EE是企业版，它只是在Java SE的基础上加上了大量的API和库。（在核心的基础上扩充）
Java ME是一个针对嵌入式设备的“瘦身版”（SE|EE 无法运行在ME上，没必要学习）

JSR规范：Java Specification Request
JCP组织：Java Community Process

RI：Reference Implementation
TCK：Technology Compatibility Kit
